Why This Static Rebuild Solves The Original Problems

The old site mixed WordPress, Gravity Forms, and SMTP/OAuth dependencies. This version keeps the public site static and fast while moving form delivery to a dedicated mail endpoint.

1

Reliable Form Routing

The homepage CTA now points to a real intake page instead of a dead anchor. Contact and dealer forms are plain HTML and can submit to a Worker or serverless function.

2

No WordPress Mail Stack

This removes Gravity Forms, WP Mail SMTP, Gmail OAuth, and database dependencies from the public-facing site entirely.

3

Deploy Anywhere

The frontend is static, so it can live on GitHub Pages, Cloudflare Pages, or Vercel. The mail endpoint can be swapped independently without redesigning the site.

Current Main Entry Points

The live WordPress site already has two working form concepts. This rebuild preserves both so you can keep your intake flows while replacing the broken delivery pipeline.

Recommended path
  • Host the static site on GitHub Pages or Cloudflare Pages.
  • Use the included Cloudflare Worker to receive form submissions.
  • Send mail through Resend with a verified domain.
  • Optionally store submissions in Worker logs, KV, D1, or a webhook later.
